# Claude Code: Your AI-Enhanced Software Engineering Assistant

Assalamualykum! Main Claude Code hoon, Anthropic ka official AI assistant jo aap ki software engineering tasks mein madad karne ke liye banaya gaya hai. Mera maqsad aap ke coding process ko smart, efficient, aur asaan banana hai. Main aap ki specifications (tafseelat) ko samajh kar aap ke tasks ko pura karta hoon.

## Main Kya Kya Kar Sakta Hoon (Meri Functionalities)

Ek Agentic AI Developer ke tor par, main aap ke liye mukhtalif qism ke tasks perform kar sakta hoon. Meri har functionality ka maqsad aap ke development workflow ko behtar banana hai:

### 1. **Code Generation & Feature Development**
*   **Kya Karta Hoon:** Aap ki zaroorat ke mutabiq naya code likh sakta hoon, chahe woh aik simple function ho, aik complex class, ya poora naya module ya feature.
*   **Kaise Karta Hoon:** Aap ki instructions ko analyse karta hoon, best practices ka khayal rakhta hoon, aur code structure ko maintain karta hoon.
*   **Example:**
    *   "Mujhe Python mein aik function likh do jo numbers ki list mein se sab se bada number dhonde."
    *   "Meri existing Node.js API mein aik naya endpoint add karo jo user profiles ko fetch kare."
    *   "Front-end par aik naya UI component banao jo user feedback form display kare."

### 2. **Bug Identification & Fixing (Debugging)**
*   **Kya Karta Hoon:** Code mein logic errors, syntax errors, ya runtime issues dhoondh kar unhein theek karta hoon.
*   **Kaise Karta Hoon:** Error messages ko analyse karta hoon, code ko line-by-line review karta hoon, aur debugging tools (agar access ho) ka istemal kar ke root cause tak pahunchta hoon.
*   **Example:**
    *   "Meri Streamlit app mein `NameError` aa raha hai, please isko theek karo." (Jaisa ke humne abhi kiya).
    *   "Yeh Python script database se data fetch karte waqt hang ho jata hai, dekho kya masla hai."

### 3. **Code Refactoring & Optimization**
*   **Kya Karta Hoon:** Code ko zyada readable, maintainable, scalable, aur performant banane mein madad karta hoon. Is mein variable/function names change karna, code duplication kam karna, ya algorithm improve karna shamil hai.
*   **Kaise Karta Hoon:** Codebase ke patterns ko samjhta hoon, performance bottlenecks identify karta hoon, aur Clean Code principles ko apply karta hoon.
*   **Example:**
    *   "Is `processData` function ko refactor karo taake yeh zyada modular aur testable ho."
    *   "`getCwd` variable ka naam `getCurrentWorkingDirectory` kar do aur poore project mein update kar do."

### 4. **Codebase Exploration & Research**
*   **Kya Karta Hoon:** Aap ke project files ko search karta hoon, specific functions ya files dhoondhta hoon, aur code ke structure aur functionality ko samjhaata hoon.
*   **Kaise Karta Hoon:** `Glob`, `Grep`, aur `Read` jaise tools ka istemal karta hoon, aur zaroorat padne par `Task` tool ke `Explore` agent ko launch karta hoon for comprehensive analysis.
*   **Example:**
    *   "Is project mein user authentication logic kahan par implement ki gayi hai?"
    *   "`src/utils` folder mein kaun kaun si utility functions hain?"

### 5. **Environment Setup & Dependency Management**
*   **Kya Karta Hoon:** Project dependencies ko manage karne mein help karta hoon (`requirements.txt`, `package.json`), virtual environments set up karta hoon, aur environment-related issues ko troubleshoot karta hoon.
*   **Kaise Karta Hoon:** `Bash` tool ka istemal karta hoon `pip install`, `npm install` jaise commands chalane ke liye, aur dependency files ko update karta hoon.
*   **Example:**
    *   "Meri `requirements.txt` file mein `streamlit` ko add karo."
    *   "Is project ke liye ek naya Python virtual environment set up karo."

### 6. **Deployment Assistance**
*   **Kya Karta Hoon:** Applications ko deployment ke liye prepare karta hoon, jaise `main.py` ko Streamlit-compatible banana ya deployment-specific configurations check karna.
*   **Kaise Karta Hoon:** Deployment platforms (jaise Streamlit Cloud) ki requirements ko samjhta hoon aur uske mutabiq code mein changes suggest ya implement karta hoon.
*   **Example:**
    *   "Meri currency converter app ko Streamlit Cloud par deploy karne ke liye tayyar karo." (Jo humne abhi kiya).

### 7. **Git & Version Control Operations**
*   **Kya Karta Hoon:** `git status`, `git add`, `git commit`, `git push` jaise basic Git commands execute karta hoon taake aap ke codebase changes track aur manage ho sakein.
*   **Kaise Karta Hoon:** Aap ki instructions par `Bash` tool ke zariye Git commands run karta hoon.
*   **Example:**
    *   "Maine jo changes kiye hain, unhein commit k करके `master` branch par push kar do."

### 8. **Task Planning & Management (Todo List)**
*   **Kya Karta Hoon:** Complex tasks ko chhote, actionable steps mein divide kar ke aik structured `Todo` list banata hoon. Har step ki progress (`pending`, `in_progress`, `completed`) track karta hoon taake aap ko clear visibility mile.
*   **Kaise Karta Hoon:** `TodoWrite` tool ka istemal karta hoon. Yeh meri planning aur execution ka aik critical hissa hai.
*   **Example:**
    *   "User registration feature implement karna hai. Iske liye database models, API endpoints, aur frontend forms banane ke steps banao."

### 9. **Interactive Clarification (Asking Questions)**
*   **Kya Karta Hoon:** Agar mujhe aap ki instructions mein koi ambiguity ho, ya koi critical decision lena ho, to main aap se `AskUserQuestion` tool ke zariye sawal poochta hoon.
*   **Kaise Karta Hoon:** Options provide karta hoon taake aap asani se choose kar sakein, ya phir aap ko custom input dene ki ijazat deta hoon.
*   **Example:**
    *   "Authentication ke liye kaun sa method prefer karenge: JWT (JSON Web Tokens) ya Session-based?\"

## Mere Advantages (Why Work with Claude Code?)

*   **Speed & Efficiency:** Tasks ko insaani developers ke muqablay mein significantly tezise complete karta hoon, khas tor par repetitive ya well-defined tasks.
*   **Accuracy & Consistency:** Code generation aur modifications mein high level ki accuracy maintain karta hoon aur consistency ensure karta hoon.
*   **Comprehensive Knowledge:** Ek vast knowledge base par trained hoon, jis se main mukhtalif programming languages, frameworks, aur best practices ko apply kar sakta hoon.
*   **Structured Task Management:** `Todo` list ke zariye aap ke tasks ko organize aur track karta hoon, jis se aap ko progress ki clear visibility milti hai.
*   **No Context Loss:** Pura conversation context yaad rakhta hoon, jis se aap ko baar baar cheezein repeat karne ki zaroorat nahi padti.

## Example: Comprehensive Prompt for a Next.js E-commerce Website

Ek professional e-commerce website develop karne ke liye, aap mujhe is tarah se detailed specifications de sakte hain. Jitni zyada tafseelat hongi, utna hi behtar main aap ke vision ko haqeeqat mein badal paunga:

---

**PROMPT EXAMPLE:**

"**Task:** Next.js mein aik complete e-commerce website develop karo. Is mein front-end (Next.js, Tailwind CSS) aur back-end (Node.js/Express ya Next.js API Routes, MongoDB database) shamil honge. Yeh responsive honi chahiye aur modern UI/UX principles follow kare.

**High-Level Features:**

1.  **User Authentication:** Sign up, Login, Logout, Password Reset, Google/GitHub OAuth (optional).
2.  **Product Management:** Admin panel se products add/edit/delete karna (name, description, price, images, categories, stock).
3.  **Product Display:**
    *   Homepage: Featured products, New arrivals.
    *   Product Listing Page: All products, filtering (by category, price range), sorting (price, popularity), pagination.
    *   Product Detail Page: Product image gallery, detailed description, add-to-cart button, related products.
4.  **Shopping Cart:** Add/remove items, update quantities, view cart summary.
5.  **Checkout Flow:**
    *   Shipping Address selection.
    *   Payment Integration: Stripe ya PayPal (sandbox mode for now).
    *   Order Confirmation.
6.  **Order Management:** User apne previous orders dekh sake, Admin saare orders manage kar sake (status update).
7.  **Search Functionality:** Products search karne ke liye search bar (fuzzy search).

**Technical Stack:**

*   **Front-end:** Next.js (Pages Router/App Router - jo modern ho), React, TypeScript (optional but preferred), Tailwind CSS (styling), Zustand/Context API (state management).
*   **Back-end:** Node.js with Express.js for a separate API, ya Next.js API Routes. Database ke liye MongoDB (Mongoose ODM ke sath).
*   **Deployment Target:** Vercel (front-end) aur Render/Fly.io (back-end API/DB).

**UI/UX Guidelines:**

*   Clean, minimalist design.
*   Mobile-first responsive approach.
*   Fast loading times (optimised images, lazy loading).
*   Consistent branding (logo, color palette - default black/white/grey, ya koi simple theme).

**Development Steps (Expected from Claude Code):**

1.  Project setup: Next.js project initialize karna, Tailwind CSS configure karna.
2.  Database setup: MongoDB connection, Mongoose schemas define karna (User, Product, Order).
3.  API Endpoints development: RESTful APIs for authentication, products, cart, orders.
4.  Front-end component development: Har feature ke liye React components banana.
5.  Integration: Front-end ko back-end APIs se connect karna.
6.  Testing: Basic unit tests for critical functions (e.g., cart logic, authentication).
7.  Documentation: `README.md` mein project setup, running instructions, aur deployment guidelines.
8.  Version Control: Regular Git commits with descriptive messages.

**Instructions:**

*   Start with the core features (Product Display, Add to Cart).
*   Har feature complete hone par mujhe update karna.
*   Agar koi architectural decision lena ho ya koi ambiguity ho, to mujhse `AskUserQuestion` tool ke zariye pooch lena.
*   Code clean aur well-commented hona chahiye.
*   `TodoWrite` tool ka istemal kar ke poore task ko steps mein divide kar ke track karte rehna.

" ---

## Aik App Banate Waqt Aap Ka Role (Human's Role: Start to End)

Jab aap mere saath mil kar koi app bana rahe honge, to aap ka role bahut zaroori hai. Aap ka kaam high-level vision aur critical decisions provide karna hai:

1.  **Conception & Requirements Definition:**
    *   **Aap ka kaam:** App ka basic idea, uska maqsad, target audience, aur core features define karna. Aap mujhe batayenge ke app kya karegi aur kaisa dikhegi (high-level pe).
    *   **Mera role:** Aap ke requirements ko samajhna aur unhein actionable technical tasks mein convert karna.

2.  **Initial Project Setup & Structure (Optional):**
    *   **Aap ka kaam:** Agar aap chahein to basic project setup khud kar sakte hain, ya mujhe directions de sakte hain.
    *   **Mera role:** Existing project structure ko analyse karna aur usmein naye components integrate karna.

3.  **Detailed Specification & Guidance:**
    *   **Aap ka kaam:** Har feature ke liye detailed specifications dena. Main jitni clear instructions receive karunga, utna hi behtar perform kar paunga.
    *   **Mera role:** Aap ki specifications ke mutabiq code likhna, bugs theek karna, ya code refactor karna.

4.  **Code Review & Validation:**
    *   **Aap ka kaam:** Mere likhe hue code ko review karna. Ensure karna ke code aap ki expectations ke mutabiq hai, secure hai, aur best practices follow karta hai. (Although main best practices follow karta hoon, final review aap ka hoga).
    *   **Mera role:** Aap ke feedback ke mutabiq code mein changes karna.

5.  **Comprehensive Testing & Quality Assurance (QA):**
    *   **Aap ka kaam:** App ki thorough testing karna. Is mein unit tests, integration tests, end-to-end tests, aur user acceptance testing (UAT) shamil hain. Aap user experience (UX) ko validate karenge.
    *   **Mera role:** Tests likhne mein help karna (agar kaha jaye), aur bugs fix karna jo testing ke dauran milte hain.

6.  **Deployment, Monitoring & Maintenance:**
    *   **Aap ka kaam:** Final deployment process ko manage karna (CI/CD setup), deployed app ki monitoring karna, aur future feature requests ya maintenance tasks ko identify karna.
    *   **Mera role:** Deployment-related tasks mein help karna aur future updates ya bug fixes ko implement karna.

## Meri Limitations (Jo Main Nahi Kar Sakta / Jo Aap Ko Hi Karna Hoga)

Main aik tool hoon, aur kuch areas hain jahan insaan ka judgment aur input na-guzir hai:

1.  **High-Level Vision & Strategy:** App ka poora vision, business goals, aur market strategy define karna sirf aap ka kaam hai. Main creative decisions nahi le sakta.
2.  **Complex Architectural Decisions:** Bahut high-level architectural choices (jaise kaun sa database use hoga, microservices ya monolith, etc.) aap ko hi karne honge. Main options suggest kar sakta hoon, lekin final decision aap ka hai.
3.  **Ethical & Moral Judgment:** Code ke ethical implications, privacy concerns, ya user data handling se mutaliq moral decisions aap ko hi lene honge. Main sirf authorized security testing mein help karta hoon, malicious activities mein nahi.
4.  **Deep User Empathy & UX Design:** Users ki feelings, unki preferences, aur unke complex behavior patterns ko samajhna, aur uske mutabiq intuitive user experience (UX) design karna, yeh insaan ka domain hai.
5.  **Comprehensive Human Testing:** Automated tests mein madad kar sakta hoon, lekin actual user scenarios mein app ko test karna aur usability feedback collect karna aap ko hi karna hoga.
6.  **Legal & Compliance Requirements:** Data protection laws (GDPR, HIPAA), industry standards, aur any other legal compliance ko ensure karna aap ki zimmedari hai.
7.  **Physical Infrastructure Management:** Servers set up karna, networking hardware manage karna, ya data centers maintain karna mere scope mein nahi hai.
8.  **Final Responsibility & Approval:** Kisi bhi software project ki ultimate responsibility aur final approval hamesha insaani developer ki hoti hai.

Main aap ka ek powerful assistant hoon, jo aap ke kaam ko bohat asaan bana sakta hai, lekin final control aur judgment hamesha aap ke paas rahega. Hum dono mil kar behtareen applications bana sakte hain!
